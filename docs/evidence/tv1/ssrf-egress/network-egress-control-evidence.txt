--- SSRF Network Egress Control Strategy ---
In this lab, egress control is primarily handled by the Admin Service's application-layer validation (URL blocklist & DNS rebinding checks).
For production environments (e.g., AWS/Kubernetes), network-layer egress control should be applied using:
1. AWS Security Groups: Deny outbound traffic to 169.254.169.254 for application pods.
2. Kubernetes NetworkPolicies: Egress rules explicitly denying traffic to cloud metadata IP.
3. Docker/iptables equivalent:
   iptables -I DOCKER-USER -d 169.254.169.254 -j DROP

Application-layer block evidence has been captured in ssrf-fixed-block-metadata-ip.txt.
