TV1 Gateway-to-backend mTLS status: IMPLEMENTED AND ENABLED IN DEFAULT COMPOSE RUNTIME.

Webhook channel mTLS status: IMPLEMENTED AND EVIDENCED.
Valid webhook client certificates are accepted, and requests without a client
certificate are rejected. This is separate from HMAC signature verification,
timestamp freshness, and replay nonce checks.

Current runtime:
The default Compose stack now fronts User/Order/Billing/Admin with Nginx mTLS
sidecars. Kong presents an internal client certificate when it connects to each
backend sidecar, and the sidecars reject direct callers without a valid client
certificate. Kong upstreams in gateway/kong.yml point to the HTTPS sidecars.

Scope boundary:
This proves Gateway-to-Backend mTLS at the backend ingress boundary.
Billing-to-Order ownership verification uses the Order mTLS sidecar with a
Billing client certificate and also requires short-lived Keycloak Client
Credentials with least-privilege service roles. The default endpoint is
https://order-mtls-proxy:8443 with lab CA verification enabled. Production can
extend this with a full service mesh/internal PKI for all east-west traffic.
